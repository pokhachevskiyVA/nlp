# nlp — обработка кардиопульмонального теста (CPET) и RR-интервалов

Инструменты для анализа нагрузочного тестирования спортсменов: сопоставление
газового анализа (CPET, «вдох за вдохом») с RR-интервалами (ритмограммой),
построение интерактивных графиков, автоматическое выделение границы
«нагрузка / восстановление» и физиологических точек 1–4.

Основной модуль — **`gas.py`**.

## Быстрый старт (Google Colab)

```python
!wget https://raw.githubusercontent.com/pokhachevskiyVA/nlp/main/gas.py -nc
import gas
gas.make()      # берёт последние загруженные .rr и .xlsx, строит и скачивает HTML
```

Нужны два файла на входе:
- `.rr` — RR-интервалы (экспорт NeuroSoft, по одному числу в строке, мс);
- `.xlsx` — отчёт CPET «вдох за вдохом» (Oxycon/Jaeger и т.п.).

## Запуск локально

```python
import gas
gas.make(rr_path="Иванов.rr",
         gas_path="20230520 Иван Иванов (CPET).xlsx",
         out_dir=".", download=False)
```

Только по RR (без газа) — см. ниже «RR-only режим».

## Что строит `gas.make()`

HTML с интерактивными графиками (Plotly), по показателю на панель, в двух
вариантах (сырой и сглаженный `_ga`):

- **VO2, VCO2, RER, VE, VE/VO2, VE/VCO2, RR, O2pulse** (кислородный пульс = VO₂/ЧСС,
  где ЧСС = 60000/RR).
- **Линия начала восстановления** — чёрная вертикаль, отделяющая нагрузку от
  восстановления (см. ниже).
- **Точки 1–4** — цветные вертикали с номерами (автоподсказки, проверяются
  глазами):
  1. лактатный / аэробный порог (VT1) — минимум VE/VO₂;
  2. ацидотический (pH) порог — консенсус изломов VCO₂/VE между точками 1 и 3;
  3. точка респираторной компенсации (RCP/VT2) — начало роста VE/VCO₂;
  4. аэробный лимит — выход VO₂ на апогей (строго внутри нагрузки, до линии
     восстановления).
- **Вторая ось абсцисс сверху — «% от МПК»** (VO₂ в % от максимума): метки
  ставятся по реальным значениям VO₂; уровни ниже покоя не рисуются.
- **Интерактив:** клик по точке рисует синхронную вертикаль и маркер на всех
  панелях и копирует координаты (время, значение, RR, %МПК) в буфер; клавиши
  Del / Esc — очистить.

Возвращает `dict` с путём к HTML, временем начала восстановления и точками.

## Начало восстановления

- **Если есть газ:** линия ставится по началу устойчивого **спада VO₂**
  (строго после пика), что гарантирует: 100% МПК и точка 4 остаются в нагрузке,
  а VO₂/VE на линии уже падают.
- **Если газа нет (только RR):** момент считается **по RR** — на сглаженной
  кривой RR находится минимум и первый устойчивый подъём после него («колено»).
  Отсчёт восстановления ведётся так, чтобы быть устойчивым к рассинхрону
  записей RR и газа.

Минуты восстановления можно задать вручную: `make(recovery_minutes=6.5)`.

## Синхронизация RR и газа

Газоанализ пишется непрерывно (предстарт/старт/паузы на подключение), а RR — строго по периодам, поэтому шкалы сдвинуты. Программа совмещает их по двум опорным точкам:

- **восстановление** — надир RR совмещается с пиком VO₂;
- **начало нагрузки** — отсчитывается по строгой RR-длительности «предстарт + старт» (по умолчанию 30+30 c, задаётся: `make(prestart_s=30, start_s=30)`).

Точки 1–4 и проценты считаются внутри истинного окна нагрузки [начало нагрузки … восстановление], а не от начала записи газа. Отключить синхронизацию: `make(align_by_recovery=False)`.

## RR-only режим (без газового анализа)

```python
import gas
gas.make_rr(rr_path="Иванов.rr", out_dir=".", download=False)
# либо gas.make() — сам перейдёт в этот режим, если рядом нет .xlsx
```

Строит RR + сглаженную кривую, линию начала восстановления и **маркеры-
кандидаты порогов** с подписями и краткой научной расшифровкой прямо в HTML:
DFA α1 = 0.75 (VT1?) и 0.5 (VT2?), «исчезновение вариабельности» RMSSD (идея
Михайлова), переломы тренда RR (идея Похачевского).

> По результатам проверки на 48 спортсменах (метки из газа) точки 1–3 по одной
> RR-ритмограмме надёжно **не** восстанавливаются (ни маркерами, ни ML — не
> лучше среднего по группе). Надёжно по RR определяются только начало
> восстановления и аэробный лимит (надир RR). Поэтому маркеры показаны как
> ориентиры «для глаза», а не как точный автоответ.

## Сглаживание

Очистка выбросов — **фильтр Хампеля** (по локальной медиане ± k·MAD в скользящем
окне): убираются артефакты, форма кривой сохраняется. Затем гауссово сглаживание
(σ=5). Прежняя чистка по глобальному порогу 1σ убирала ~треть точек и искажала
форму — заменена.

## Ключевые функции `gas.py`

| Функция | Назначение |
|---|---|
| `make(...)` | Полный отчёт по паре RR + газ. |
| `make_rr(...)` | Отчёт только по RR (+ маркеры-кандидаты). |
| `detect_recovery_start_vo2(...)` | Начало восстановления по спаду VO₂. |
| `detect_recovery_start_curve(...)` | Начало восстановления по RR («колено»). |
| `detect_points(...)` | Авто-поиск точек 1–4 по газовым кривым. |
| `rr_candidate_markers(...)` | RR-маркеры порогов (DFA α1, RMSSD, изломы). |
| `smooth_curve / hampel_filter` | Сглаживание кривых. |
